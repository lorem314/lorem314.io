## (网页)样式简史

CSS 在网络刚刚开始逐渐流行的 1994 年被首次提案。在当时，浏览器将调整样式的权力完全交给了用户，如 Mosaic 浏览器中的首选项(Preferences)中，用户可以调整字体系列、大小和颜色。网站作者所能做的，就是将一段文字标为段落，或是标题，或是其他特殊样式的元素类型。用户全权控制网页的外观。

正是在这种环境下，CSS 诞生了。它的目标是向网站的作者提供一种简单、声明式的样式语言，这种语言对网站作者来说是灵活的，但更重要的是，它同时位网站作者和浏览器用户提供了相似修改页面样式的功能。通过级联(cascade)的方式，这些样式可以被组合，并生成优先级，这样一来网站作者和浏览器用户就都拥有发言权 —— 但浏览器用户有最终发言权。

工作进展迅速，于 1996 年底，CSS1 已经完成。在新成立的 CSS 工作组推进 CSS2 的同时，浏览器也在努力以可交互的方式实施 CSS1。尽管 CSS 的每小块代码看上去都很简单，但组合起来却能呈现出惊人的复杂表现。尽管早期的实现中也有一些不幸的食物，如盒子模型实现中的差异。这些问题可能会毁坏 CSS，但幸运的是，一些高明的提议被提出并得到了实施，浏览器开始协调一致。几年之内，由于互操作性的提高和备受瞩目的发展，Wired 杂志和基于 CSS 的 CSS Zen Garden 的重新设计，CSS 开始流行起来。

<blockquote>

[CSS Zen Garden](http://www.csszengarden.com/)，该网站主要用于展示通过基于 CSS 设计可以达到的视觉效果。

</blockquote>

不过在此之前，CSS 工作小组于 1998 年初完成了 CSS2 的规范。CSS2 的工作一完成，马上就又开始了 CSS3 的工作，同时也包括更加规范版本的 CSS2，CSS2.1 版本。为了与时代精神保持一致，CSS3 被构建为一系列的(理论上)独立模块，而非单一的整体规范。这种方法反映了当时有效的 XHTML 规范，出于类似的原因也被设计为多个模块。

模块化 CSS3 进行开发的原因，是因为这样的话，每个模块都可以按照自己的节奏进行开发，对于特别关键（或流行）的模块可以遵循 W3C 的进度轨道推进，而不会被其他模块阻碍。确实，事实证明情况确实如此。2012 年初期，有 3 个 CSS 模块达到了完整推荐状态 —— CSS Color Level 3，CSS Namespaces 和 Selector Level 3。同时，有 7 个模块处于候选推荐状态，还有其他十多个处于草稿期的不同阶段的模块。得益于模块化开发，已完成的 3 个模块不必等待其他模块达到同样的开发完整度，便可直接加入完整规范。

模块化优势的另一面是很难谈论单一的 CSS3 规范。没有这样的东西，也不可能有。即使所有的模块都达到了 Level 3，2016 年底的时候还没有，Selector Level 4 已经在着手进行。我们会将其成为 CSS4 吗？那么所有仍在发挥作用的 CSS3 的特性呢？或是 Grid 布局，当时还未达到 Level 1 阶段。

## 元素

元素(element)是组成文档的基础。在 HTML 中，很容易辨别出常用的元素，如 p、table、span、a、和 div。文档中的每个元素都在页面呈现中起着各自的作用。

### 替换和非替换元素

尽管 CSS 离不开元素，并不是所有的元素的创建方式都是相同的。例如，img 和 p 不是同一类型的元素，span 和 div 也不是。在 CSS 中，元素一般分为两个种类：可替换(replaced)和非可替换(non-replaced)。

#### 替换元素

替换元素是指，元素的内容将被替换，不是直接来自文档内容的东西替换。最熟悉的就是 img 元素，会被替换为文档外部的一张图片。事实上，img 没有实际的内容，如下：

```html
<img src="howdy.gif" />
```

这段标记只包含元素名称和一个属性。该元素将展示 src 属性指向的路径的图片，若该路径有效，则对应的图片将被展示在文档中，否则，浏览器将什么都不显示或者显示一个损坏的图片的占位符。

类似的，input 元素也是替换元素，根据 type 属性的不同，可被一个小型圆圈、选中框或是文本输入框替换。

#### 非替换元素

大多数的 HTML 元素都是非替换元素。其内容通常就是元素标签中所包含的内容。例如，

```html
<span>hi there</span>
```

就是一个非替换元素，文本 hi there 将由浏览器原封不动的展示出来。段落标签 p、标题 h1 到 h6、表格 table、列表 ul、ol 和其他大多数 HTML 标签都是如此。

### 元素显示的职能

除了替换和非替换元素之外，CSS 中还可以根据另外一种方法，将元素分成两类：**块级元素**(block-level)和**行内元素**(inline-level)。除此之外还有很多种显示类型，这两个是最基础的，也是大多数其他显示类型所指的。块级和行内元素常在页面中出现，对于经常与 HTML 打交道的人并不陌生。图 1-1 展示了这两种元素。

<BorderBlock>

<h1>一级标题（块级）</h1>

该段落(p)元素是一个块级元素。接下来的这段粗体字是**行内元素，在必要时会换行。**行内元素外的内容仍是块级元素的一部分。行内元素中的内容<i>好比这几个文字</i>属于行内元素。

</BorderBlock>

#### 块级元素

块级元素在默认情况下占满父元素的全部宽度，不能有其他元素在其左右。换句话说，其在元素的盒子模型前后设置了换行。HTML 中最常见的块级元素是 p 和 div。替换元素可以是块级元素，但通常他们不是。

列表中的 li 元素是块级元素中的一个例外。他们除了占据整行，还在元素之前添加了标记 —— 通常情况下 ul 列表下的 li 元素的标记是一个实心圆点，ol 列表的是数字 —— 该标记是附着在元素盒子上的。除了这些标记外，在其他方面列表元素和其他合块级元素没有区别。

#### 行内元素

行内元素，顾名思义是在某行之内创建元素盒子，不会迫使换行。常见的行内元素是 a 元素。其他的还有 strong 和 em。这些元素不会在其前后添加换行，故可以在不干扰其他元素展示的情况下出现在其他元素中。

需要注意的是，在 HTML 中将块级元素放到行内元素中会引发一些奇怪的现象，同时也是不推荐的做法。但在 CSS 中并没有这一规定，因为我们可以修改 display 属性。

至于现在，我们只关注块级和行内两个值。考虑如下 HTML：

```html
<body>
  <p>这是一个有着<em>一个行内元素</em>在其中的段落。</p>
</body>
```

我们有两个块级元素，body 和 p，还有一个行内元素 em。根据 HTML 规范，em 元素可以出现在 p 元素中，反之则不可。
